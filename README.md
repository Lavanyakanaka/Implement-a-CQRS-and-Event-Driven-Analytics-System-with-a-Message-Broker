# Implement a CQRS and Event-Driven Analytics System with a Message Broker

## Overview

This project demonstrates the implementation of the Command Query Responsibility Segregation (CQRS) pattern combined with an Event-Driven Architecture using RabbitMQ as the message broker. The system separates write operations from read operations and maintains analytics views through asynchronous event processing.

## Architecture

The system consists of the following components:

### Command Service

* Handles write operations.
* Receives order creation requests.
* Stores data in the write database.
* Publishes events to RabbitMQ.

### RabbitMQ

* Acts as the message broker.
* Distributes events generated by the Command Service.
* Decouples producers and consumers.

### Consumer Service

* Consumes events from RabbitMQ.
* Processes order events asynchronously.
* Updates analytics and read models.

### Query Service

* Handles read operations.
* Retrieves data from the read database.
* Provides optimized analytics queries.

### Databases

* **Write Database:** Stores transactional data.
* **Read Database:** Stores denormalized analytics views for fast querying.

## Setup Instructions

### Prerequisites

* Docker
* Docker Compose
* Git

### Clone Repository

```bash
git clone https://github.com/Lavanyakanaka/Implement-a-CQRS-and-Event-Driven-Analytics-System-with-a-Message-Broker.git

cd Implement-a-CQRS-and-Event-Driven-Analytics-System-with-a-Message-Broker
```

### Start Services

```bash
docker compose up --build
```

Or run in detached mode:

```bash
docker compose up -d --build
```

### Verify Containers

```bash
docker ps
```

Expected services:

* command-service
* query-service
* consumer-service
* rabbitmq
* write-db
* read-db

## Workflow

1. Client sends order request to Command Service.
2. Command Service writes data to Write DB.
3. Command Service publishes an event to RabbitMQ.
4. Consumer Service receives the event.
5. Consumer Service updates analytics/read models.
6. Query Service serves analytics data from Read DB.

## CQRS Benefits

* Separation of read and write responsibilities.
* Independent scaling of read and write workloads.
* Improved maintainability.
* Better performance for analytics queries.
* Event-driven extensibility.
